# 研究生招生信息管理系统 — 数据库设计报告

---

## 一、数据库概述

- **数据库名称**：`postgraduate_admission`
- **数据库系统**：MySQL 8.0+
- **字符集**：UTF-8（`utf8mb4`）
- **存储引擎**：InnoDB（支持事务和外键约束）
- **设计范式**：第三范式（3NF）

## 二、数据表设计

### 2.1 专业字典表（major_dict）

**用途**：存储所有招生专业的基础信息，包括专业代码、名称和计划招生人数。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| `major_code` | VARCHAR(10) | PRIMARY KEY | 专业代码 |
| `major_name` | VARCHAR(50) | NOT NULL | 专业名称 |
| `planned_inside` | INT | DEFAULT 0 | 计划内招生数 |
| `planned_outside` | INT | DEFAULT 0 | 计划外招生数 |

```sql
CREATE TABLE `major_dict` (
    `major_code` VARCHAR(10) PRIMARY KEY COMMENT '专业代码',
    `major_name` VARCHAR(50) NOT NULL COMMENT '专业名称',
    `planned_inside` INT DEFAULT 0 COMMENT '计划内招生数',
    `planned_outside` INT DEFAULT 0 COMMENT '计划外招生数'
) COMMENT='专业字典';
```

---

### 2.2 考生档案表（candidate_profile）

**用途**：存储所有报名考生的个人信息与报考志愿，是系统的核心基础表。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| `exam_id` | VARCHAR(15) | PRIMARY KEY | 考号 |
| `name` | VARCHAR(30) | NOT NULL | 考生姓名 |
| `gender` | CHAR(1) | — | 性别（男/女） |
| `age` | INT | — | 年龄 |
| `political_status` | VARCHAR(20) | — | 政治面貌 |
| `is_fresh_graduate` | BOOLEAN | — | 是否应届毕业生 |
| `education` | VARCHAR(20) | — | 学历（本科/硕士/博士） |
| `origin` | VARCHAR(50) | — | 来源地 |
| `target_major` | VARCHAR(10) | FK → major_dict | 报考专业代码 |
| `category` | VARCHAR(20) | — | 报考类别（计划内/计划外） |

```sql
CREATE TABLE `candidate_profile` (
    `exam_id` VARCHAR(15) PRIMARY KEY COMMENT '考号',
    `name` VARCHAR(30) NOT NULL COMMENT '姓名',
    `gender` CHAR(1) COMMENT '性别',
    `age` INT COMMENT '年龄',
    `political_status` VARCHAR(20) COMMENT '政治面貌',
    `is_fresh_graduate` BOOLEAN COMMENT '是否应届',
    `education` VARCHAR(20) COMMENT '学历',
    `origin` VARCHAR(50) COMMENT '来源地',
    `target_major` VARCHAR(10) COMMENT '报考专业代码',
    `category` VARCHAR(20) COMMENT '报考类别',
    FOREIGN KEY (`target_major`) REFERENCES `major_dict`(`major_code`)
) COMMENT='考生档案';
```

---

### 2.3 初试成绩表（first_test_score）

**用途**：存储考生的初试各科成绩，总分通过虚拟列自动计算，保证数据一致性。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| `exam_id` | VARCHAR(15) | PRIMARY KEY, FK | 考号 |
| `politics` | DECIMAL(5,2) | — | 政治成绩（0-100） |
| `foreign_lang` | DECIMAL(5,2) | — | 外语成绩（0-100） |
| `major_basis` | DECIMAL(5,2) | — | 专业基础科目成绩（0-100） |
| `total_first` | DECIMAL(6,2) | VIRTUAL（自动计算）| 初试总分 |

```sql
CREATE TABLE `first_test_score` (
    `exam_id` VARCHAR(15) PRIMARY KEY COMMENT '考号',
    `politics` DECIMAL(5,2) COMMENT '政治成绩',
    `foreign_lang` DECIMAL(5,2) COMMENT '外语成绩',
    `major_basis` DECIMAL(5,2) COMMENT '专业基础科目',
    `total_first` DECIMAL(6,2) GENERATED ALWAYS AS
        (politics + foreign_lang + major_basis) VIRTUAL COMMENT '初试总分',
    FOREIGN KEY (`exam_id`) REFERENCES `candidate_profile`(`exam_id`)
) COMMENT='初试成绩';
```

> **设计要点**：`total_first` 使用 `GENERATED ALWAYS AS ... VIRTUAL` 虚拟列，总分由数据库自动计算，无需应用层手动求和，避免数据不一致问题。

---

### 2.4 复试成绩表（second_test_score）

**用途**：存储考生的复试各环节成绩，包括专业科目笔试、面试和上机测试。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| `exam_id` | VARCHAR(15) | PRIMARY KEY, FK | 考号 |
| `major_test` | DECIMAL(5,2) | — | 复试专业科目成绩（0-100） |
| `interview` | DECIMAL(5,2) | — | 面试成绩（0-100） |
| `computer_test` | DECIMAL(5,2) | — | 上机成绩（0-100） |
| `total_second` | DECIMAL(6,2) | VIRTUAL（自动计算）| 复试总分 |

```sql
CREATE TABLE `second_test_score` (
    `exam_id` VARCHAR(15) PRIMARY KEY COMMENT '考号',
    `major_test` DECIMAL(5,2) COMMENT '复试专业科目',
    `interview` DECIMAL(5,2) COMMENT '面试成绩',
    `computer_test` DECIMAL(5,2) COMMENT '上机成绩',
    `total_second` DECIMAL(6,2) GENERATED ALWAYS AS
        (major_test + interview + computer_test) VIRTUAL COMMENT '复试总分',
    FOREIGN KEY (`exam_id`) REFERENCES `candidate_profile`(`exam_id`)
) COMMENT='复试成绩';
```

---

### 2.5 录取名单表（admission_list）

**用途**：存储最终被录取的考生信息，记录最终初试成绩、复试成绩和综合总分。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| `exam_id` | VARCHAR(15) | PRIMARY KEY, FK | 考号 |
| `admitted_major` | VARCHAR(10) | FK → major_dict | 录取专业代码 |
| `final_first_score` | DECIMAL(6,2) | — | 最终初试成绩 |
| `final_second_score` | DECIMAL(6,2) | — | 最终复试成绩 |
| `total_score` | DECIMAL(6,2) | — | 综合总分 |

```sql
CREATE TABLE `admission_list` (
    `exam_id` VARCHAR(15) PRIMARY KEY COMMENT '考号',
    `admitted_major` VARCHAR(10) COMMENT '录取系别(专业代码)',
    `final_first_score` DECIMAL(6,2) COMMENT '最终初试成绩',
    `final_second_score` DECIMAL(6,2) COMMENT '最终复试成绩',
    `total_score` DECIMAL(6,2) COMMENT '综合总分',
    FOREIGN KEY (`exam_id`) REFERENCES `candidate_profile`(`exam_id`),
    FOREIGN KEY (`admitted_major`) REFERENCES `major_dict`(`major_code`)
) COMMENT='录取名单';
```

## 四、表关系说明

| 关系 | 主表 | 从表 | 类型 | 说明 |
|------|------|------|------|------|
| 专业 → 考生 | major_dict | candidate_profile | 1:N | 一个专业可被多个考生报考 |
| 考生 → 初试成绩 | candidate_profile | first_test_score | 1:1 | 每个考生有一份初试成绩 |
| 考生 → 复试成绩 | candidate_profile | second_test_score | 1:1 | 每个考生有一份复试成绩 |
| 考生 → 录取名单 | candidate_profile | admission_list | 1:0..1 | 考生可能被录取或不录取 |
| 专业 → 录取名单 | major_dict | admission_list | 1:N | 录取到一个专业 |

## 五、索引设计

| 表名 | 索引名 | 字段 | 类型 | 用途 |
|------|--------|------|------|------|
| candidate_profile | PRIMARY | exam_id | 主键 | 考生唯一标识 |
| candidate_profile | fk_target_major | target_major | 外键 | 按专业查询考生 |
| first_test_score | PRIMARY | exam_id | 主键 | 考生初试唯一记录 |
| second_test_score | PRIMARY | exam_id | 主键 | 考生复试唯一记录 |
| admission_list | PRIMARY | exam_id | 主键 | 录取考生唯一记录 |
| admission_list | fk_admitted_major | admitted_major | 外键 | 按专业查录取 |

## 六、数据库部署说明

### 6.1 创建数据库

```sql
CREATE DATABASE postgraduate_admission
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 6.2 初始化步骤

1. 登录 MySQL：`mysql -u root -p`
2. 创建数据库：执行上述建库语句
3. 导入表结构：`source database/schema.sql;`
4. 导入测试数据（如有提供 seed.sql）

### 6.3 连接配置

后端 `application.yml` 中的配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/postgraduate_admission
      ?useUnicode=true
      &characterEncoding=utf-8
      &useSSL=false
      &serverTimezone=Asia/Shanghai
    username: root
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

## 七、统计查询设计

系统通过 MyBatis XML Mapper 实现了以下统计查询：

| 统计项 | Mapper方法 | 说明 |
|--------|------------|------|
| 各科成绩概览 | `scoreOverview()` | 平均分、最高分、最低分 |
| 各科及格率 | `passRate()` | ≥60分为及格 |
| 初试分数段分布 | `firstScoreDistribution()` | <200, 200-250, 250-300, >300 |
| 复试分数段分布 | `secondScoreDistribution()` | 同上分段 |
| 各专业录取人数 | `countByMajor()` | JOIN major_dict 统计 |
| 录取生来源分布 | `countByOrigin()` | GROUP BY origin |
| 录取生学历分布 | `countByEducation()` | GROUP BY education |
| 计划vs实际招生 | `planVsActual()` | 对比 planned 与 actual |
