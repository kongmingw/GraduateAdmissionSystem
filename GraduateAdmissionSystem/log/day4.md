### 4. 初试筛选与复试综合筛选（核心业务逻辑）

**初试筛选 API：**
- `GET /api/statistics/screening?year=2026`
- 查询当年分数线 → 遍历所有初试成绩 → 逐个比对四条线（政治≥单科线、外语≥单科线、专业基础≥单科线、初试总分≥总分线）
- 返回合格列表（进复试）和不合格列表（含具体淘汰原因，如"政治45<50"）

**复试综合筛选（前端实现）：**
- 点分数线「筛选」按钮 → 先调用初试筛选 API → 再查复试成绩 → 计算综合总分 → 对比录取总分线
- 分为「可录取考生」（综合≥录取线）和「未达标/未复试」（综合不够或还没复试）
- 未达标项红色标出差额，未复试项灰色标记

**录取分数线检查：**
- 更新 `AdmissionListController.admit()`：录取时自动查 `score_line`，综合总分不达录取总分线则拒绝，返回具体差额

## 二、遇到的问题与解决方案

### 问题 1：`ScoreStatisticsMapper.xml` 解析失败
- **现象**：后端启动时 MyBatis 报 `SAXParseException`，XML 第27行解析错误
- **根因**：SQL 中 `politics < 60` 的 `<` 被 XML 解析器当作标签起始符号
- **解决**：将所有含 `<` 比较运算符的 SQL 语句用 `<![CDATA[...]]>` 包裹

### 问题 2：端口 8080 被占用
- **现象**：二次启动后端时 `PortInUseException`
- **解决**：`netstat -ano | findstr :8080` 查 PID → `taskkill` 杀掉旧进程

### 问题 3：MySQL 中文乱码
- **现象**：导入 `seed.sql` 时中文报 `Incorrect string value`
- **解决**：mysql 命令加 `--default-character-set=utf8mb4` 参数

### 问题 4：专业空代码提交
- **现象**：专业代码未填也能保存，且删除时 URL 路径 `/delete/` 为空导致 404
- **根因**：前端未校验必填字段，空字符串作主键
- **解决**：SQL 直接清理脏数据，前端加非空校验

### 问题 5：el-select 选中后不显示文本
- **现象**：报考专业下拉框选中后输入框无文字，仅显示箭头或叉号
- **解决**：选项内显式写 `{{ m.majorName }}` + 固定宽度 220px
