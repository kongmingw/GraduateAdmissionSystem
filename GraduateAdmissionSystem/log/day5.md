## 一、完成内容

### 1. 表单校验强化

**考生管理：**
- 引入 Element Plus 表单校验规则（`el-form :rules`），8 个必填字段：考号、姓名、性别、政治面貌、学历、来源地、报考专业、报考类别
- 未填字段自动红框标记 + 红色提示文字，关闭对话框自动清除校验状态
- 政治面貌从文本框改为下拉选择（中共党员/中共预备党员/共青团员/群众/其他党派）

**专业字典：**
- 保存时校验专业代码和名称不能为空

---

### 2. 招生名额限制

**录取名额上限：**
- `AdmissionListController.admit()` 增加名额检查逻辑：
  - 统计目标专业当前已录取的计划内/计划外人数
  - 计划内考生录取时检查是否超出 `planned_inside`
  - 计划外考生录取时检查是否超出 `planned_outside`
  - 总人数超出计划总数也拒绝录取
  - 拒绝时返回具体数字（"该专业计划内名额已满（计划30人，已录取30人）"）

**招生数修改限制：**
- `MajorDictController.update()` 增加校验：修改招生数时，新数值不能低于该专业当前已录取人数
- 例：CS001 已录2人计划内 → 不能将 planed_inside 改为1

- 专业删除：外键约束阻止时提示"该专业有考生报考或录取记录，无法删除"
- 区分用户主动取消和接口报错（`e !== 'cancel'`）

### 4. 统计分析可视化

使用纯 CSS 实现图表，无需第三方库：

- **4 个彩色概览卡片**：考生总数、已录取、招生专业、初试均分
- **及格率进度条**（`el-progress`）：政治、外语、专业基础三科并排，红绿分色
- **成绩概览卡片**：每科最高分（绿）、平均分（蓝）、最低分（红）
- **初试/复试分数段柱状图**：CSS `div` + 动态 `width` 实现
- **计划vs实际招生柱状图**：灰色计划底 + 绿色实际录取重叠
- **来源/学历分布环形饼图**：CSS `conic-gradient` 实现，8 色调色板 + 图例
- **各科及格率环形饼图**：绿色填满比例 + 灰色空白
