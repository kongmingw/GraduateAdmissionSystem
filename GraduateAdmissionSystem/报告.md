# 摘    要

研究生招生信息管理是高等教育管理中的重要环节，涉及考生报名、初试与复试成绩管理、录取分数线设定以及录取后的统计分析等多个阶段。传统的纸质或分散的系统管理方式存在信息孤岛、数据冗余和处理效率低下等问题。针对上述问题，本文设计并实现了一套基于Spring Boot和Vue.js的研究生招生信息管理系统。系统采用前后端分离架构，后端基于Spring Boot+MyBatis框架实现RESTful API，前端采用Vue 3+Element Plus构建用户界面，数据库采用MySQL进行数据持久化。系统涵盖专业字典管理、考生档案管理、初试成绩管理、复试成绩管理、分数线管理、录取管理和统计分析七大功能模块，支持考生从报名到录取的全流程管理，并应用了事务管理确保数据一致性。经测试，系统运行稳定，功能完整，可有效提升研究生招生管理工作的效率与准确性。

# Abstract

Graduate admission information management is a critical aspect of higher education administration, encompassing multiple stages including candidate registration, initial and re-examination score management, admission score line configuration, and post-admission statistical analysis. Traditional paper-based or fragmented digital management approaches suffer from information silos, data redundancy, and low processing efficiency. To address these challenges, this paper designs and implements a Graduate Admission Information Management System based on Spring Boot and Vue.js. The system adopts a front-end and back-end separation architecture, with the backend built on the Spring Boot+MyBatis framework providing RESTful APIs, the frontend developed using Vue 3+Element Plus for the user interface, and MySQL employed for data persistence. The system comprises seven functional modules: major dictionary management, candidate profile management, first-test score management, second-test score management, score line management, admission management, and statistical analysis, supporting the complete workflow from candidate registration to admission. Transaction management is applied to ensure data consistency. Testing demonstrates that the system operates stably with complete functionality, effectively improving the efficiency and accuracy of graduate admission management.

**Key Words：** Graduate Admission; Information Management System; Spring Boot; Vue.js; MyBatis

# 1. 引言

## 1.1 系统背景

研究生招生工作是高等教育体系中的重要组成部分，其管理过程通常分为三个阶段：第一阶段是考生报名，考生提交个人简历与报考信息，形成考生档案；第二阶段是初试与复试，招生人员需要设定分数线、录入考试成绩、根据分数线筛选合格考生进入复试环节，并最终确定录取名单；第三阶段是录取后的统计分析，包括对录取生的成绩、年龄、来源分布等方面进行多维度的统计分析，以及计划招生数与实际招生数的对比分析。

目前，许多高校的研究生招生管理工作仍然依赖纸质文档或Excel电子表格进行管理，存在以下突出问题：各阶段数据分散在不同文件中，难以统一管理和追溯；数据统计需要大量手工计算，效率低下且容易出错；缺乏统一的分数线自动筛选机制，人工比对耗时耗力；招生计划与实际录取的对比分析不便直观呈现。因此，开发一套信息化、自动化的大学生研究生招生信息管理系统具有重要的现实意义。

## 1.2 系统开发的意义

从业务层面来看，本系统实现了研究生招生全流程的信息化管理，将考生报名、成绩录入、分数线筛选、录取决策和统计分析等环节串联起来，形成一个完整的数据闭环。招生人员可以在统一的平台上完成所有管理工作，大幅减少重复录入和数据传递中的错误。

从技术层面来看，系统采用B/S架构与前后端分离的开发模式，后端以RESTful API的形式提供数据服务，便于后续扩展和对接其他系统。前端采用响应式单页应用，操作流畅，用户体验良好。数据库层面通过外键约束和事务管理保证数据的一致性和完整性，如删除考生时自动检查其是否存在关联成绩记录，录取时通过事务确保多表写操作的原子性。

从学习层面来看，本项目综合运用了Spring Boot后端开发、MyBatis持久层框架、Vue 3前端开发、MySQL数据库设计等多门课程知识，同时涉及事务管理、构造器注入、代码规范化等工程实践，对提升软件开发综合能力具有重要价值。

# 2. 相关技术研究

## 2.1 Spring Boot

Spring Boot是Spring生态系统中的一个开源框架，旨在简化基于Spring的Java应用程序的创建和部署过程。其核心设计理念是"约定优于配置"，通过自动配置（Auto-Configuration）机制，开发者无需编写繁琐的XML配置文件即可快速搭建生产级的Spring应用。Spring Boot内嵌了Tomcat、Jetty等Servlet容器，使得应用可以以JAR包的形式独立运行，极大地简化了部署流程。

在本系统中，Spring Boot作为后端核心框架，承担了以下职责：通过@RestController注解构建RESTful API接口，为前端提供数据服务；通过@Transactional注解声明事务边界，确保录取等关键操作中多表写操作的数据一致性；通过CorsConfig实现跨域请求支持，解决前后端分离开发中的跨域问题；通过application.yml统一管理数据源、MyBatis、服务器端口等配置信息。

## 2.2 Vue.js

Vue.js是一款用于构建用户界面的渐进式JavaScript框架。其核心库专注于视图层，采用响应式数据绑定和组合式API，使得开发者可以通过声明式渲染和组件化开发高效地构建交互丰富的单页应用。Vue 3引入了Composition API（组合式API），以setup函数和响应式引用（ref、reactive）为核心，相比Vue 2的Options API，在逻辑复用、类型推断和代码组织上均有显著提升。

在本系统中，前端采用Vue 3+Vite+Element Plus+Vue Router+Axios技术栈。Element Plus作为UI组件库，提供了表格、表单、对话框、标签页、统计面板等丰富的组件，加速了界面开发。Vue Router实现了客户端路由，支持考生管理、成绩管理、录取管理等7个页面的无刷新导航。Axios封装了HTTP请求，通过Vite代理将/api请求转发至后端8080端口，解决了开发阶段的跨域问题。

## 2.3 MyBatis

MyBatis是一款优秀的Java持久层框架，通过XML或注解将Java对象与SQL语句映射起来，消除了几乎所有的JDBC代码和手动参数设置。与Hibernate等全自动ORM框架不同，MyBatis是半自动化的，开发者可以直接编写和优化SQL语句，对复杂查询具有更强的控制力。MyBatis支持动态SQL、结果映射、缓存等特性，在对SQL性能要求较高的场景中表现出色。

在本系统中，MyBatis通过Spring Boot Starter进行集成。每个数据库表对应一个Mapper接口和一个XML映射文件。接口定义方法签名，XML映射文件编写SQL语句。通过application.yml中配置的`map-underscore-to-camel-case: true`，数据库下划线命名的字段（如exam_id）自动映射为Java的驼峰命名属性（如examId），减少了大量冗余的ResultMap配置。在统计分析模块中，MyBatis的动态SQL和CDATA标签被用于编写含比较运算符的聚合查询语句。

# 3. 需求分析

根据研究生招生信息管理业务的实际需求，系统需包含以下核心功能：

- **专业字典管理**：招生人员可维护专业信息，包括专业代码、专业名称、计划内招生数和计划外招生数。修改招生数时需校验新数值不得低于该专业当前已录取人数，防止数据矛盾。

- **考生档案管理**：支持考生信息的录入、查询、修改和删除。考生信息包括考号、姓名、性别、年龄、政治面貌、是否应届、学历、来源地、报考专业和报考类别。支持按考号、姓名或报考专业进行搜索过滤。删除考生时需检查是否存在关联成绩或录取记录。

- **初试成绩管理**：录入考生的政治成绩、外语成绩和专业基础科目成绩。初试总分由数据库虚拟列自动计算（politics+foreign_lang+major_basis），避免前后端计算不一致的问题。界面需提示尚未录入成绩的考生列表。

- **复试成绩管理**：录入复试专业科目成绩、面试成绩和上机成绩。仅初试合格的考生才有资格录入复试成绩，后端需进行双重校验。复试总分同样由数据库虚拟列自动计算。

- **分数线管理**：设定各年度的招生分数线，包括政治单科线、外语单科线、专业基础单科线、初试总分线和录取总分线。支持对分数线的新增、修改、删除和按年份查询。

- **初试筛选**：根据当年分数线自动判定考生复试资格。考生需同时满足三条单科线（各科成绩≥对应单科线）和初试总分线（总分≥初试总分线），任何一项不达标均判定为不合格，并给出具体淘汰原因（如"政治45<50"）。

- **复试综合筛选**：在初试合格考生中，结合复试成绩计算综合总分（初试总分+复试总分），与录取总分线比较，分为可录取和未达标两类，未达标项标出具体差额。

- **录取管理**：录取时自动获取考生初试和复试成绩，计算综合总分，检查是否达到录取总分线。同时检查目标专业的计划内/计划外招生名额是否已满，超出名额则拒绝录取。支持取消录取操作。

- **统计分析**：对录取数据进行多维度统计分析，包括各专业录取人数、录取生来源分布、学历分布、计划与实际招生对比。对成绩数据进行分析，包括各科平均分/最高分/最低分、各科及格率/不及格率、初试和复试总分分数段分布。全部图表使用纯CSS实现，不依赖第三方图表库。

# 4. 总体设计

## 4.1 架构设计

系统采用B/S三层架构与前后端分离模式，整体架构如下所示：

```
┌─────────────────────────────────────────────────────────┐
│                    浏览器（Browser）                      │
│         Vue 3 + Element Plus + Axios                     │
│                  http://localhost:5173                    │
└─────────────────────┬───────────────────────────────────┘
                      │ HTTP /api/*
                      │ (Vite Dev Proxy)
┌─────────────────────┴───────────────────────────────────┐
│               Spring Boot 后端（Server）                   │
│                  http://localhost:8080                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐      │
│  │Controller│→│ Service  │→│  Mapper (MyBatis) │      │
│  │  (7个)   │  │   (7个)  │  │      (7个)        │      │
│  └──────────┘  └────┬─────┘  └────────┬─────────┘      │
│                     │ @Transactional   │                  │
└─────────────────────┼─────────────────┼──────────────────┘
                      │                 │
┌─────────────────────┴─────────────────┴──────────────────┐
│                  MySQL 数据库（Data）                      │
│           postgraduate_admission (6张表)                  │
│  major_dict │ candidate_profile │ first_test_score        │
│  second_test_score │ admission_list │ score_line          │
└──────────────────────────────────────────────────────────┘
```

前端Vue 3应用在5173端口运行，开发阶段通过Vite的代理配置将`/api`前缀的请求转发到后端8080端口。后端Spring Boot应用采用分层架构：Controller层负责接收HTTP请求和参数校验，Service层封装业务逻辑并声明事务边界，Mapper层通过MyBatis与MySQL数据库交互。数据库选用MySQL，创建6张业务表，表之间通过外键约束保证引用完整性。

## 4.2 功能模块设计

系统共划分为七大功能模块，各模块功能描述如下：

- **专业字典管理模块**：负责维护所有招生专业的基本信息和招生计划数。支持专业的增删改查操作，修改招生数时自动校验已录取人数，防止将计划数调低至已录取人数以下导致逻辑矛盾。

- **考生档案管理模块**：负责考生基本信息的全生命周期管理。提供8项必填字段的表单校验（考号、姓名、性别、政治面貌、学历、来源地、报考专业、报考类别），支持文本搜索和专业筛选的组合过滤。删除考生时若存在关联的成绩或录取记录，由数据库外键约束拦截并向前端返回友好提示。

- **初试成绩管理模块**：负责政治、外语、专业基础三门初试科目成绩的录入与维护。提供可搜索的考生下拉选择框和待录入考生列表，选择考生后自动检测是新增还是编辑模式。总分由数据库GENERATED ALWAYS AS虚拟列计算，确保数据准确性。

- **复试成绩管理模块**：负责复试专业科目、面试、上机三门复试科目成绩的录入。待录入列表仅展示初试合格的考生，后端保存时再次校验考生初试过线状态，进行双重保障。

- **分数线管理模块**：负责各年度分数线的设定和维护。支持分数线的新增、编辑和删除。集成初试筛选功能：点击筛选按钮或表格行，自动根据选定年份的分数线对全体考生进行初试资格判定。同时集成复试综合筛选：在初试合格基础上，结合复试成绩与录取总分线进行二次筛选。

- **录取管理模块**：负责录取名单的管理。采用三区布局：上部展示已录取名单，中部展示可录取考生（初试过线+复试完成+综合总分达线），下部展示未达标考生（综合总分不够线，标红差额）。录取时后端自动获取初试和复试成绩，计算综合总分，检查录取分数线和招生名额限制，所有检查通过后在同一事务中完成录取数据写入。

- **统计分析模块**：负责录取数据和成绩数据的多维度统计与可视化展示。包含4个概览统计卡片、各科及格率进度条、成绩概览表格、分数段分布柱状图、计划vs实际对比图和来源/学历分布环形饼图。全部图表使用纯CSS技术实现（conic-gradient、动态width等），不依赖ECharts等第三方图表库，减少了前端依赖体积。

## 4.3 数据库设计

此次系统使用的数据库为MySQL数据库，对数据进行持久层的存储与处理，确保了数据的安全存储和持久性。经过综合设计与考虑，并针对实际需求的需要，设置了6张业务表。各表之间通过外键约束维护数据引用完整性。

### 4.3.1 专业字典表（major_dict）

专业字典表用于存储招生专业的基本信息，包括专业代码、专业名称和计划内外招生数。该表是考生档案和录取名单的外键引用表，为整个招生流程提供专业基础数据。

## 5.2 专业字典管理

专业字典管理页面提供招生专业的维护功能。表格展示专业代码、专业名称、计划内招生数和计划外招生数四列数据。支持按专业代码或名称进行搜索过滤。

专业字典的关键业务逻辑在于修改招生数时的校验：系统会统计该专业当前已录取的计划内和计划外考生人数，若新设的计划数低于已录取人数，则拒绝修改。此外，在录取考生时，系统也会检查目标专业的名额是否已满，防止超额录取。

```java
// 后端：专业字典更新时的名额限制校验（MajorDictService.java）
@Transactional
public void update(MajorDict majorDict) {
    MajorDict existing = majorDictMapper.findByCode(majorDict.getMajorCode());
    if (existing == null) {
        throw new RuntimeException("专业不存在");
    }
    // 统计该专业已录取的计划内和计划外人数
    int admittedInside = 0, admittedOutside = 0;
    for (AdmissionList a : admissionListMapper.findAll()) {
        if (majorDict.getMajorCode().equals(a.getAdmittedMajor())) {
            CandidateProfile cp = candidateProfileMapper.findByExamId(a.getExamId());
            if (cp != null) {
                if ("计划内".equals(cp.getCategory())) admittedInside++;
                else admittedOutside++;
            }
        }
    }
    if (majorDict.getPlannedInside() < admittedInside) {
        throw new RuntimeException("计划内招生数不能低于已录取人数(" + admittedInside + ")");
    }
    if (majorDict.getPlannedOutside() < admittedOutside) {
        throw new RuntimeException("计划外招生数不能低于已录取人数(" + admittedOutside + ")");
    }
    majorDictMapper.update(majorDict);
}
```

## 5.4 复试成绩管理

复试成绩管理页面结构与初试成绩管理类似，但有一个关键区别：待录入列表仅展示通过初试筛选的合格考生，初试不合格的考生不会出现在复试成绩录入候选中。这一过滤逻辑通过调用后端的screening API获取当年合格考生名单来实现。

后端在保存复试成绩时进行双重校验，确保只有初试合格的考生才能录入复试成绩：

```java
// 后端：复试成绩录入时的初试过线校验（SecondTestScoreService.java）
@Transactional
public void add(SecondTestScore score) {
    // 1. 检查是否有初试成绩
    FirstTestScore firstScore = firstTestScoreMapper.findByExamId(score.getExamId());
    if (firstScore == null) {
        throw new RuntimeException("该考生无初试成绩，不能录入复试成绩");
    }
    // 2. 检查初试是否过线
    ScoreLine line = scoreLineMapper.findByYear(String.valueOf(Year.now().getValue()));
    if (line != null) {
        if (firstScore.getPolitics() < line.getPoliticsLine()
            || firstScore.getForeignLang() < line.getForeignLangLine()
            || firstScore.getMajorBasis() < line.getMajorBasisLine()
            || firstScore.getTotalFirst() < line.getTotalFirstLine()) {
            throw new RuntimeException("该考生初试未过线，不能录入复试成绩");
        }
    }
    secondTestScoreMapper.insert(score);
}
```

## 5.5 分数线管理与筛选

分数线管理页面是本系统的核心业务页面之一，集成了分数线CRUD和双阶段筛选功能。表格展示所有年度的分数线记录，每行包含年份、三条单科线和两条总分线。操作列提供编辑、删除和筛选三个按钮。

**初试筛选功能**是本系统的关键业务逻辑。调用screening API后，后端遍历所有有初试成绩的考生，逐一比对四条线（政治≥单科线、外语≥单科线、专业基础≥单科线、初试总分≥总分线），四项全部满足才判定为"合格"，任何一项不满足即为"不合格"并记录具体淘汰原因。筛选结果以Tabs形式展示：合格标签页和不合格标签页。

```java
// 后端：初试筛选核心逻辑（StatisticsService.java）
public Map<String, Object> screening(String year) {
    ScoreLine line = scoreLineMapper.findByYear(year);
    if (line == null) throw new RuntimeException("该年份未设定分数线");

    List<FirstTestScore> scores = firstTestScoreMapper.findAll();
    List<Map<String, Object>> qualified = new ArrayList<>();
    List<Map<String, Object>> disqualified = new ArrayList<>();

    for (FirstTestScore s : scores) {
        List<String> reasons = new ArrayList<>();
        if (s.getPolitics() < line.getPoliticsLine())
            reasons.add("政治" + s.getPolitics() + "<" + line.getPoliticsLine());
        if (s.getForeignLang() < line.getForeignLangLine())
            reasons.add("外语" + s.getForeignLang() + "<" + line.getForeignLangLine());
        if (s.getMajorBasis() < line.getMajorBasisLine())
            reasons.add("专业基础" + s.getMajorBasis() + "<" + line.getMajorBasisLine());
        if (s.getTotalFirst() < line.getTotalFirstLine())
            reasons.add("初试总分" + s.getTotalFirst() + "<" + line.getTotalFirstLine());

        Map<String, Object> entry = new HashMap<>();
        entry.put("examId", s.getExamId());
        entry.put("politics", s.getPolitics());
        entry.put("foreignLang", s.getForeignLang());
        entry.put("majorBasis", s.getMajorBasis());
        entry.put("totalFirst", s.getTotalFirst());

        if (reasons.isEmpty()) {
            entry.put("status", "合格");
            qualified.add(entry);
        } else {
            entry.put("status", "不合格");
            entry.put("reasons", reasons);
            disqualified.add(entry);
        }
    }
    // 返回结果包含：年份、分数线、合格/不合格数量和明细
    Map<String, Object> result = new HashMap<>();
    result.put("year", year);
    result.put("scoreLine", line);
    result.put("qualifiedCount", qualified.size());
    result.put("disqualifiedCount", disqualified.size());
    result.put("qualified", qualified);
    result.put("disqualified", disqualified);
    return result;
}
```

**复试综合筛选**在后端实现（通过`GET /api/statistics/admission-screening`接口）：系统先调用初试筛选获取合格考生，再查询复试成绩和考生信息，对每位合格考生计算综合总分（初试+复试），与录取总分线比较，分为"可录取考生"（综合≥线）和"未达标/未复试"（综合<线或尚无复试成绩）两类，未达标项标出具体差额分数。该逻辑封装在StatisticsService.admissionScreening()方法中：

```java
// 后端：复试综合筛选（StatisticsService.java）
public Map<String, Object> admissionScreening(String year) {
    // 先做初试筛选
    Map<String, Object> firstResult = screening(year);
    ScoreLine line = (ScoreLine) firstResult.get("scoreLine");
    List<Map<String, Object>> qualified = (List<Map<String, Object>>) firstResult.get("qualified");

    // 查复试成绩和考生信息
    var secondMap = new HashMap<String, Double>();
    secondTestScoreMapper.findAll().forEach(s -> secondMap.put(s.getExamId(), s.getTotalSecond()));
    var nameMap = new HashMap<String, String>();
    candidateProfileMapper.findAll().forEach(c -> nameMap.put(c.getExamId(), c.getName()));

    List<Map<String, Object>> admitted = new ArrayList<>();
    List<Map<String, Object>> failed = new ArrayList<>();

    for (Map<String, Object> q : qualified) {
        String examId = (String) q.get("examId");
        double totalFirst = ((Number) q.get("totalFirst")).doubleValue();
        Double totalSecond = secondMap.get(examId);

        double total = totalFirst + (totalSecond != null ? totalSecond : 0);
        if (totalSecond == null) {
            entry.put("status", "未复试"); failed.add(entry);
        } else if (total >= line.getAdmissionTotalLine()) {
            entry.put("status", "可录取"); admitted.add(entry);
        } else {
            entry.put("status", "不达标"); failed.add(entry);
        }
    }
    return Map.of("line", line.getAdmissionTotalLine(), "admitted", admitted, "failed", failed);
}
```

## 5.6 录取管理

录取管理页面采用三区布局，直观展示录取全貌。上部为已录取名单，中部为可录取考生列表（复试已完成且综合总分达线），下部为未达标考生列表（综合总分不够线，红色标出差额）。

录取操作时，后端AdmissionListService.admit()方法承担了复杂的业务逻辑，共有5道校验关卡，全部通过后在同一事务中写入录取记录：

```java
// 后端：录取核心逻辑——5道校验关卡（AdmissionListService.java）
@Transactional
public String admit(AdmissionList request) {
    String examId = request.getExamId();

    // ① 检查是否已被录取
    if (admissionListMapper.findByExamId(examId) != null) {
        throw new RuntimeException("该考生已在录取名单中");
    }

    // ② 获取初试成绩
    FirstTestScore firstScore = firstTestScoreMapper.findByExamId(examId);
    if (firstScore == null) {
        throw new RuntimeException("该考生无初试成绩，无法录取");
    }

    // ③ 获取复试成绩
    SecondTestScore secondScore = secondTestScoreMapper.findByExamId(examId);
    if (secondScore == null) {
        throw new RuntimeException("该考生无复试成绩，无法录取");
    }

    // ④ 计算综合总分，检查录取总分线
    double totalScore = firstScore.getTotalFirst() + secondScore.getTotalSecond();
    ScoreLine line = scoreLineMapper.findByYear(String.valueOf(Year.now().getValue()));
    if (line != null && line.getAdmissionTotalLine() != null) {
        if (totalScore < line.getAdmissionTotalLine()) {
            throw new RuntimeException("综合总分(" + totalScore +
                ")未达到录取总分线(" + line.getAdmissionTotalLine() + ")，无法录取");
        }
    }

    // ⑤ 检查招生名额（计划内/计划外/总计）
    CandidateProfile candidate = candidateProfileMapper.findByExamId(examId);
    String admittedMajor = request.getAdmittedMajor();
    if (candidate != null && admittedMajor != null) {
        MajorDict major = majorDictMapper.findByCode(admittedMajor);
        if (major != null) {
            int currentCount = 0, currentPlanNei = 0, currentPlanWai = 0;
            for (AdmissionList a : admissionListMapper.findAll()) {
                if (admittedMajor.equals(a.getAdmittedMajor())) {
                    currentCount++;
                    CandidateProfile cp = candidateProfileMapper.findByExamId(a.getExamId());
                    if (cp != null) {
                        if ("计划内".equals(cp.getCategory())) currentPlanNei++;
                        else currentPlanWai++;
                    }
                }
            }
            int totalPlan = major.getPlannedInside() + major.getPlannedOutside();
            if (currentCount >= totalPlan) {
                throw new RuntimeException("该专业招生名额已满（计划" + totalPlan +
                    "人，已录取" + currentCount + "人）");
            }
            if ("计划内".equals(candidate.getCategory())
                && currentPlanNei >= major.getPlannedInside()) {
                throw new RuntimeException("该专业计划内名额已满（计划" +
                    major.getPlannedInside() + "人，已录取" + currentPlanNei + "人）");
            }
            if ("计划外".equals(candidate.getCategory())
                && currentPlanWai >= major.getPlannedOutside()) {
                throw new RuntimeException("该专业计划外名额已满（计划" +
                    major.getPlannedOutside() + "人，已录取" + currentPlanWai + "人）");
            }
        }
    }

    // 通过全部校验，写入录取记录
    request.setFinalFirstScore(firstScore.getTotalFirst());
    request.setFinalSecondScore(secondScore.getTotalSecond());
    request.setTotalScore(totalScore);
    admissionListMapper.insert(request);

    return "录取成功，综合总分：" + totalScore;
}
```

录取管理的5道校验关卡分别为：查重（①）、查初试成绩（②）、查复试成绩（③）、查录取分数线（④）、查招生名额（⑤），层层把关确保录取数据的正确性和业务规则的严格执行。

另外，系统提供了候选列表API（`GET /api/admission/candidates`），由后端AdmissionListService.getCandidates()方法统一处理筛选逻辑，自动遍历所有考生，筛选出有初试+有复试+未被录取的考生，并按录取总分线分为合格与不合格两组：

```java
// 后端：获取待录取候选列表（AdmissionListService.java）
public Map<String, Object> getCandidates() {
    Set<String> admittedIds = new HashSet<>();
    admissionListMapper.findAll().forEach(a -> admittedIds.add(a.getExamId()));

    // 构建初试和复试成绩映射
    Map<String, Double> firstMap = new HashMap<>();
    firstTestScoreMapper.findAll().forEach(f -> firstMap.put(f.getExamId(), f.getTotalFirst()));
    Map<String, Double> secondMap = new HashMap<>();
    secondTestScoreMapper.findAll().forEach(s -> secondMap.put(s.getExamId(), s.getTotalSecond()));

    List<Map<String, Object>> qualified = new ArrayList<>();
    List<Map<String, Object>> unqualified = new ArrayList<>();

    candidateProfileMapper.findAll().forEach(c -> {
        if (admittedIds.contains(c.getExamId())) return;
        Double totalFirst = firstMap.get(c.getExamId());
        Double totalSecond = secondMap.get(c.getExamId());
        if (totalFirst == null || totalSecond == null) return;

        double totalScore = totalFirst + totalSecond;
        Map<String, Object> entry = Map.of("examId", c.getExamId(),
            "name", c.getName(), "totalScore", totalScore);
        if (totalScore >= admissionLine) qualified.add(entry);
        else unqualified.add(entry);
    });
    return Map.of("admissionLine", admissionLine, "qualified", qualified, "unqualified", unqualified);
}
```

前端AdmissionList.vue的loadCandidates方法通过单次API调用即可获取完整的候选列表数据。

## 5.7 统计分析

统计分析页面汇总展示所有招生数据，分为录取统计和成绩统计两大板块。所有图表均使用纯CSS和Element Plus组件实现，不依赖ECharts等第三方图表库，减少了前端依赖体积。

页面顶部为4个彩色概览统计卡片（使用el-statistic组件），分别展示考生总数、已录取人数、招生专业数和初试平均分，各卡片以不同颜色区分，一目了然。

成绩分析区展示三科及格率（≥60分为及格）的进度条（el-progress组件）：政治、外语、专业基础三科并排排列，进度条颜色根据及格率高低自动呈现红绿分色。成绩概览区以卡片形式展示各科的最高分（绿色标记）、平均分（蓝色标记）和最低分（红色标记）。

分数段分布采用CSS div柱状图实现，使用动态计算的百分比作为div的width属性，叠加在统一的灰色背景容器上，实现了柱状图的视觉效果。初试总分分为四段（<200、200-249、250-299、≥300），复试总分同样分为四段（<200、200-230、231-260、>260），各段人数和占比一目了然。

计划vs实际招生对比图采用双色叠加柱状图实现：灰色底柱表示计划招生总数，绿色覆盖柱表示实际录取人数，通过并排对比直观展示各专业的招生完成情况。

来源分布和学历分布使用CSS conic-gradient实现环形饼图，8色调色板为各类别着色，配合图例标注。实现方式如下：


统计数据的后端查询采用MyBatis编写聚合SQL，以分数段分布查询为例：

```xml

<select id="firstScoreDistribution" resultType="map">

        SUM(CASE WHEN total_first &lt;  200 THEN 1 ELSE 0 END) AS lt200,
```

# 6. 总结与展望

本文设计并实现了一套基于Spring Boot和Vue.js的研究生招生信息管理系统。系统采用前后端分离的B/S架构，后端基于Spring Boot+MyBatis构建RESTful API，前端基于Vue 3+Element Plus构建用户界面，MySQL作为数据持久化方案。系统实现了专业字典管理、考生档案管理、初试成绩管理、复试成绩管理、分数线管理、录取管理和统计分析共七大功能模块，覆盖了研究生招生信息管理从考生报名、初试复试到录取统计的完整业务流程。

在技术实现方面，本系统具有以下特点：（1）数据库层面采用虚拟列（GENERATED ALWAYS AS VIRTUAL）自动计算初试和复试总分，从根源上避免数据不一致问题；（2）业务层面通过@Transactional注解确保录取等关键操作中多表写操作的原子性；（3）通过@RestControllerAdvice实现全局异常处理，将RuntimeException统一转换为Result.error响应，消除Controller层重复的try-catch代码，使业务代码更加简洁清晰；（4）录取操作中集成了5道校验关卡（查重、查初试、查复试、查分数线、查名额），层层把关确保数据质量；（5）复试综合筛选和录取候选计算等复杂业务逻辑封装在后端Service层，前端仅负责API调用和数据展示，职责清晰、架构合理；（6）筛选模块自动生成具体淘汰原因（如"政治45<50"），为招生决策提供精确依据；（7）统计分析使用纯CSS实现图表可视化（conic-gradient环形饼图、动态width柱状图），无需额外引入ECharts等大型图表库，减少了前端依赖体积；（8）前后端均采用构造器注入（@RequiredArgsConstructor）的设计模式，代码规范、可测试性强。

本系统仍存在一些可改进之处。在功能方面，可增加用户登录与权限控制功能，支持多角色操作（如招生人员、审核人员、管理员）。在性能方面，统计分析中的多表聚合查询可在数据量增长后考虑引入Redis缓存以提升响应速度。在安全性方面，可引入HTTPS协议和敏感数据（如考生个人信息）的加解密存储。此外，筛选逻辑目前采用循环逐条比对的方式，在考生规模较大时可考虑引入多线程并行处理以提升效率。以上改进方向将作为后续迭代的优化目标。
